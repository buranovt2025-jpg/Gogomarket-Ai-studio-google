# Техническое задание (ТЗ) — Social Marketplace

## 1. Общее описание
Современный маркетплейс (аналог OLX) с элементами социальной сети (Stories, Reels, Explore Grid). Приложение ориентировано на мобильные устройства и публикацию в App Store / Google Play.

## 2. Ключевые фишки
- **Stories**: Короткие исчезающие посты от продавцов.
- **Reels**: Вертикальная лента видео-обзоров товаров.
- **Explore Grid**: Динамическая сетка товаров в стиле Instagram Explore.
- **Интеграция логистики**: Яндекс Доставка, Самовывоз, Собственная служба.

## 3. Дизайн и Интерфейс
- **Стиль**: Dark Luxury / Tech (Instagram Style).
- **Цветовая схема**: Глубокий черный (#050505), акцентный изумрудный или белый для текста.
- **Типографика**: Современный гротеск (Inter/Space Grotesk).
- **Атмосфера**: Иммерсивная, акцент на медиа-контенте.

## 4. Функциональные модули
### 4.1. Главный экран (Explore)
- **Структура**: Masonry Grid (Алгоритмический хаос).
- **Контент**: Смешанный (фото 1x1, видео-превью 1x2).
- **Stories**: Горизонтальная лента в верхней части экрана.
- **Интерактив**: Автовоспроизведение видео при скролле (без звука).

### 4.2. Видео-лента (Reels)
- **Формат**: Полноэкранное вертикальное видео (9:16).
- **Интерфейс (Overlay)**: Название товара, цена, кнопка "Купить" (Quick Buy).
- **Социальные функции**: Лайк, Добавить в избранное, Кнопка быстрого перехода в чат с продавцом.
- **Навигация**: Вертикальный свайп для переключения между роликами.

### 4.3. Создание контента (Listing Creation)
- **Процесс**: AI-First (Умная загрузка).
- **Логика**: Пользователь загружает медиа (фото/видео) -> Gemini анализирует контент -> Автогенерация заголовка, описания, категории и тегов.
- **Редактирование**: Пользователь проверяет данные и устанавливает цену.


### 4.4. Мессенджер (Chat)
- **Стиль**: Telegram-style (простой, быстрый, интуитивный).
- **Функционал**: Личные сообщения, обмен фотографиями, возможность делиться ссылками на товары.
- **Технология**: Real-time сообщения через WebSockets.
- **Назначение**: Свободное общение между пользователями, обсуждение деталей сделки и обмен информацией.

### 4.5. Профиль и Социальные связи
- **Модель**: Полноценная соцсеть (Followers & Verification).
- **Функции**: Подписки на продавцов, уведомления о новых Stories/Reels, система отзывов и оценок.
- **Верификация**: "Синие галочки" для проверенных магазинов и частных лиц.
- **Витрина**: Профиль как персональный бренд с архивом Reels и активными Stories.

### 4.6. Поиск и Обнаружение (Search & Discovery)
- **Модель**: Гибридная система (AI + Visual + Classic).
- **AI-Поиск**: Семантический поиск на естественном языке (Gemini).
- **Визуальный поиск**: Поиск по фотографиям и скриншотам.
- **Классический поиск**: Поиск по ключевым словам и детальная система фильтров (цена, категория, локация).

### 4.7. Монетизация (Monetization)
- **Модель**: Комбинированная (Комиссия + Продвижение).
- **Комиссия**: Процент с каждой сделки через "Безопасную сделку" (Escrow).
- **Продвижение (Boost)**: Платное поднятие Reels и Stories в ленте Explore и результатах поиска.
- **Подписки для бизнеса**: Расширенные профили для магазинов с аналитикой и кастомным оформлением.

### 4.8. Уведомления (Notifications)
- **Модель**: Smart Push (Умные уведомления).
- **Типы**: Транзакционные (заказы, чаты), Социальные (новые Stories/Reels от подписок, лайки), Маркетинговые (снижение цен, персональные рекомендации).
- **Настройки**: Гибкое управление типами уведомлений в профиле пользователя.

## 5. Логика и Безопасность
### 5.1. Основные сценарии (User Flows)
- **Покупка**: Поиск -> Выбор доставки -> Оплата (Escrow) -> Трекинг -> Подтверждение -> Выплата продавцу.
- **Продажа**: Загрузка медиа -> AI-анализ (Gemini) -> Установка цены -> Публикация -> Управление заказом.
- **Общение**: Прямой чат по товару -> Согласование -> Переход к оформлению.

### 5.2. Система логистики и платежей
- **Методы**: Яндекс Доставка (API), Самовывоз, Собственная служба доставки.
- **Оплата**: Полная предоплата через приложение (Escrow/Безопасная сделка).
- **Процесс**: Деньги замораживаются -> Вызов курьера/подготовка к самовывозу -> Подтверждение получения -> Перевод денег продавцу.
- **Трекинг**: Интеграция карт для отслеживания курьеров и выбора пунктов выдачи.

## 6. Технологический стек
- **Frontend**: React, Tailwind CSS, Framer Motion.
- **Backend**: Node.js, Express.
- **Database**: SQLite (better-sqlite3).
- **AI**: Google Gemini API (@google/genai).
- **Mobile**: Capacitor (для публикации в App Store/Google Play).
